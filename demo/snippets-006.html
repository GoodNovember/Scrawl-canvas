Test purpose

  • Apply text-based effect snippets to a series of <h2> header elements
  • Edit the header elements - the effect should update in real-time
  • Resize the browser window - the effect should update in real-time

Note 1: only tested on Latin-based (alphabetical left-to-right) fonts.

Known issue 1: the HTML and canvas-based text will break at different points - this is because the canvas text is measuring lines including the space at the end of each word, while browsers remove the space at the end of each line. TODO: Needs to be fixed

Known issue 2: canvas text assumes that hyphenated words are single words and won't break on the hyphen, unlike browsers. TODO: consider how to handle all the various different ways that browsers break text into lines.

Known issue 3: canvas text cannot handle multiline line breaks (the return key). TODO: decide if we want to fix this and, if yes, fix it.
